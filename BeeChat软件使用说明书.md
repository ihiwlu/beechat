# BeeChat 即时通讯软件使用说明书

---

## 一、软件概述

### 1.1 软件名称

BeeChat即时通讯软件

### 1.2 软件版本

V3

### 1.3 开发目的

开发一个稳定、安全、易用的在线互动聊天系统。

### 1.4 面向行业/领域

普通互联网用户

### 1.5 开发环境

| 项目 | 说明 |
|------|------|
| 开发操作系统 | Windows 11 |
| 开发工具 | Cursor |
| 编程语言 | Python |
| 客户端框架 | wxPython 4.0 |
| 数据库 | MySQL 5.7 |
| 网络通信 | Socket套接字 |

### 1.6 运行平台与支撑环境

| 项目 | 要求 |
|------|------|
| 运行平台/操作系统 | Windows 10及以上 |
| Python版本 | Python 3.7及以上 |
| 数据库 | MySQL 5.7及以上 |
| 网络要求 | 有网络连接 |

### 1.7 源程序量

本软件源程序总量为 **6275行** 代码，包含服务端和客户端两部分：

- **服务端代码**：包含15个Python源文件，涵盖7个独立服务器模块和数据库管理工具
- **客户端代码**：包含9个Python源文件，涵盖8个用户界面和1个程序入口

### 1.8 软件主要功能

BeeChat是一款基于C/S（客户端/服务器）架构的即时通讯软件，系统功能涵盖 **用户模块、好友模块、聊天模块、文件模块及邮件验证模块**，具体如下：

#### （1）用户模块
- **用户注册**：支持用户名、密码、邮箱注册，系统通过SMTP发送5位数字邮箱验证码进行身份验证，验证码有效期为5分钟
- **用户登录**：安全的用户名/密码登录验证，登录成功后自动连接各服务器
- **密码重置**：通过邮箱验证码重置密码，系统校验邮箱信息并发送5位数字验证码

#### （2）好友模块
- **添加好友**：通过用户名搜索并发送好友请求
- **好友请求处理**：管理收到的好友请求，支持接受或拒绝操作
- **好友列表**：自动刷新好友列表（每10秒），实时显示好友信息
- **备注修改**：支持为好友设置个性化备注名，方便识别和管理

#### （3）聊天模块
- **私聊功能**：支持与好友进行一对一实时文字聊天
- **实时传输**：基于Socket套接字的实时双向消息传输
- **消息历史**：自动保存聊天记录到MySQL数据库，支持查看历史消息
- **离线消息**：支持离线消息存储和展示，对方上线后可查看

#### （4）文件模块
- **文件上传**：支持向好友发送任意类型文件
- **文件下载**：支持接收并下载好友发送的文件
- **流式传输**：采用流式文件传输技术，支持大文件传输
- **进度展示**：下载过程中实时展示传输进度
- **文件管理**：查看共享文件记录，包括文件名、大小、上传时间等信息

#### （5）邮件验证模块
- **验证码发送**：通过SMTP邮件服务发送5位数字验证码
- **频率限制**：设置验证码发送频率限制，防止滥用
- **有效期管理**：验证码有效期为5分钟，超时自动失效
- **自动清理**：后台定时自动清理过期的验证码记录

### 1.9 技术特点

1. **wxPython图形界面设计**：客户端使用wxPython 4.0框架进行UI设计，提供直观友好的图形用户界面
2. **套接字数据传递**：使用Socket套接字进行客户端与服务器之间的数据传递，实现实时双向通信
3. **客户端/服务器（C/S）模式**：采用经典的客户端/服务器架构，服务器端负责业务逻辑处理和数据管理，客户端提供用户交互界面
4. **多服务器模块化架构**：每个功能模块独立部署在专用服务器上（登录、注册、忘记密码、聊天、好友列表、文件传输、验证码共7个服务器），便于维护和扩展
5. **MySQL数据持久化**：所有用户数据、好友关系、聊天记录、文件信息、验证码等均持久化存储在MySQL数据库中
6. **多线程并发处理**：服务器端使用多线程处理并发客户端请求，客户端使用子线程处理网络通信避免阻塞UI线程
7. **流式文件传输**：文件上传和下载采用流式传输技术，支持断线重试（最多5次），确保文件传输可靠性

## 二、软件安装与启动

### 2.1 安装前准备

在安装和运行BeeChat之前，请确保以下环境已准备就绪：

1. 操作系统为 Windows 10 或以上版本
2. 已安装 Python 3.7 或以上版本
3. 已安装 MySQL 5.7 或以上版本，且MySQL服务已启动
4. 计算机已连接到网络

### 2.2 安装Python依赖库

打开命令提示符（CMD）或PowerShell，执行以下命令安装所需依赖：

```bash
pip install wxPython pymysql
```

其中：
- **wxPython**：客户端图形界面框架
- **pymysql**：服务器端MySQL数据库驱动

### 2.3 数据库配置

MySQL数据库配置信息位于 `server/config.py` 文件中，默认配置如下：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| host | localhost | 数据库服务器地址 |
| user | root | 数据库用户名 |
| password | pwd@123 | 数据库密码 |
| port | 3306 | 数据库端口 |
| charset | utf8mb4 | 字符集 |
| database | test | 数据库名称 |

> 如需修改数据库配置，请编辑 `server/config.py` 文件中对应的值。

### 2.4 创建数据库

在MySQL命令行客户端或图形化管理工具（如Navicat、phpMyAdmin）中执行以下命令创建数据库：

```sql
CREATE DATABASE test DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

> 首次运行服务器时，程序会自动创建所需的全部数据表，无需手动建表。

### 2.5 启动服务器端

**步骤1：进入服务器目录**

打开命令提示符，进入 `server` 文件夹：

```bash
cd server
```

执行以下命令启动所有服务器：


或直接使用mana.py快捷方式

## 四、系统架构说明

### 4.1 整体架构

BeeChat采用C/S（客户端/服务器）架构，系统整体分为三层：

1. **表示层（客户端）**：使用wxPython 4.0框架构建图形用户界面，处理用户交互操作，通过Socket套接字与服务器通信
2. **业务逻辑层（服务器）**：7个独立服务器进程分别处理登录、注册、忘记密码、聊天、好友管理、文件传输、验证码等业务逻辑
3. **数据层（MySQL数据库）**：使用MySQL 5.7数据库持久化存储用户数据、好友关系、聊天记录、文件信息和验证码

### 4.2 服务器模块

系统共包含7个独立服务器模块，每个模块监听不同端口，独立运行：

| 序号 | 服务器名称 | 文件名 | 端口 | 功能描述 |
|------|-----------|--------|------|----------|
| 1 | 登录服务器 | loginserver.py | 8888 | 处理用户登录验证 |
| 2 | 注册服务器 | registerserver.py | 8889 | 处理用户注册（含验证码校验） |
| 3 | 找回密码服务器 | forgotserver.py | 8890 | 处理邮箱验证和密码重置 |
| 4 | 聊天服务器 | chatserver.py | 8891 | 处理实时消息的接收和转发 |
| 5 | 好友列表服务器 | friendlistserver.py | 8892 | 处理好友关系管理及聊天记录持久化 |
| 6 | 文件传输服务器 | fileserver.py | 8893 | 处理文件上传下载 |
| 7 | 验证码服务器 | mail_id.py | 12123 | 处理邮箱验证码发送与验证 |

## 六、常见问题与解决方法

### 6.1 数据库连接失败

**问题描述**：启动服务器时提示数据库连接失败。

**解决方法**：
1. 检查MySQL服务是否已启动
2. 检查 `server/config.py` 中的数据库配置信息是否正确（用户名、密码、端口等）
3. 确认数据库 `test` 已创建：`CREATE DATABASE test DEFAULT CHARACTER SET utf8mb4;`一般会自动创建数据库

### 6.2 客户端无法连接服务器

**问题描述**：登录时提示"连接服务器失败"。

**解决方法**：
1. 确保服务器已通过 `python mana.py` 成功启动，7个服务均运行中
2. 检查防火墙设置，确保端口8888-8893和12123未被阻止
3. 尝试重新启动服务器

### 6.3 收不到验证码

**问题描述**：注册或找回密码时点击"发送验证码"后收不到邮件。

**解决方法**：
1. 确认邮箱地址输入正确
2. 检查邮箱的垃圾邮件/垃圾箱文件夹
3. 确保验证码服务器（mail_id.py，端口12123）已启动
4. 如提示"发送频繁"，请等待5分钟后再试

### 6.4 消息发送失败

**问题描述**：聊天时点击发送后消息无法发出。

**解决方法**：
1. 确保聊天服务器（chatserver.py，端口8891）已启动
2. 确保好友列表服务器（friendlistserver.py，端口8892）已启动（消息持久化依赖此服务）
3. 检查网络连接是否正常
4. 尝试重新登录

### 6.5 文件传输失败

**问题描述**：文件无法上传或下载。

**解决方法**：
1. 确保文件传输服务器（fileserver.py，端口8893）已启动
2. 确保选择了有效的文件（文件路径不为空）
3. 检查网络连接，下载失败时会自动重试最多5次
4. 确保服务器 `server/uploads/` 目录存在且有写入权限

### 6.6 好友列表不显示

**问题描述**：登录后好友列表为空或无法刷新。

**解决方法**：
1. 确保好友列表服务器（friendlistserver.py，端口8892）已启动
2. 点击"刷新"按钮手动刷新
3. 检查数据库中好友关系数据是否存在

## 七、安全说明

### 7.1 账号安全
- 密码建议使用字母、数字、符号的组合，提高安全性
- 不要使用过于简单的密码
- 如忘记密码，可通过注册邮箱重置

### 7.2 邮箱安全
- 注册邮箱建议使用私人邮箱，确保能正常收发邮件
- 不要将邮箱验证码告知他人
- 验证码5分钟有效期，过期后需重新获取

### 7.3 聊天安全
- 不要在聊天中发送敏感个人信息
- 谨慎对待陌生人的好友请求

### 7.4 网络安全
- 建议在安全的网络环境下使用
- 确保服务器端运行在可信环境中
